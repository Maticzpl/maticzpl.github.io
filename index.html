  • User interface

Devlog 07.02.2021

Using OpenGL wasn't the best choice but I learned a lot.

I still dont have enough knowledge about OpenGL. Developing a system around a technology you don't fully understand brings many unexpected issues.

Renderable.h was supposed to bring OpenGL to a bit higher level so i dont have to manage VBOs and VAOs manually. With every bit of new knowledge about OpenGL i found new issues in this file. They mainly come to adding new data and trying not to create a VBO for every single object.

Renderable was developed before writing the UI. With the current setup I would have to create a new VBO for every new part of the UI, which is inefficient.

Another thing is optimizing data updates. Currently i assemble an array of the datatypes I want to sent to the GPU. For the voxel part filling that array with data from the simulation takes a lot of time (size of the array is 2097152 * the datatype im sending). If I want to optimise that I would have to check which voxels are hidden behind other voxels and then don't add them. Doing that on CPU side also doesn't sound good.

I'm also concidering just rewriting the whole thing so I dont have to fill in a new array at all.
